# box2bitable

基于豆包大模型和飞书多维表格的智能鞋盒标签识别与盘点工具（微信小程序版）。

## 1. 项目简介
`box2bitable` 旨在通过 AI 技术简化仓库盘点与业务数据录入流程。用户通过微信小程序拍照上传鞋盒标签图片，系统调用豆包大模型自动识别（支持一图多条记录），经人工复核后按模块规则聚合/生成字段并写入飞书多维表格；同时提供库存查询能力，可按货号查询尺码与数量分布。

## 2. 核心功能
- **两大板块**
  - **数据录入**：采购 / 销售 / 库存 三个模块。
  - **数据查询**：库存查询（按货号查询尺码与数量）。
- **智能识别**：集成豆包大模型，支持一图多条标签记录识别。
- **人工复核**：移动端友好界面，支持修正识别结果；销售模块支持补充金额/支付方式/备注等字段。
- **写入与口径**
  - **采购/库存**：按 `SKU_Code` 聚合并写入飞书，数量口径为“写入时累加（upsert 累加数量）”。
  - **销售**：明细流水表，每次写入均为“新增一条记录”（不使用 `SKU_Code` 作为唯一键）。
- **图片同步**：上传图片到飞书获取 `file_token`，再写入多维表格的附件字段（同一张图片可在多条记录中复用同一 `file_token`）。

## 3. 技术栈
- **前端**：微信小程序原生开发 (WXML, WXSS, JS)
- **后端**：Node.js + Express
- **数据库/存储**：Supabase (PostgreSQL + Storage)
- **AI 模型**：豆包 (Doubao) 大模型 API
- **表格服务**：飞书多维表格 (Feishu Bitable) API

## 5. 快速开始
1. 安装依赖并启动后端
   - `cd server && npm install`
   - `npm run dev`
2. 使用微信开发者工具打开 `miniprogram/` 目录
3. 开发者工具本地调试建议
   - 勾选“**不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书**”
4. 飞书图片写入排查（可选）
   - 设置 `FEISHU_DEBUG=true` 输出飞书上传与写入的入参/出参日志（token 默认脱敏）
   - 设置 `FEISHU_DEBUG_SHOW_TOKENS=true` 输出 token 明文（仅建议本地排查使用）
