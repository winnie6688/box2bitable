## 1. 产品概述

box2bitable 是一个面向仓库/门店现场操作的微信小程序，用户上传鞋盒标签图片后，系统调用豆包大模型解析标签信息（支持一图多条记录），前端进行人工复核，后端按业务模块规则聚合并写入飞书多维表格；同时提供库存查询能力，输入货号即可查询该货号下各尺码的数量分布。

## 2. 信息架构（两大板块）

### 2.1 数据录入

数据录入包含三个模块：
- 采购
- 销售
- 库存

三个模块使用同一种图片来源（鞋盒标签），差异体现在“解析字段”“复核字段”“写入表格与写入口径”。

### 2.2 数据查询

当前提供库存查询：
- 输入货号（item_no）
- 返回该货号下的尺码列表与数量信息
- 数据源为飞书库存多维表格（实时查询）

## 3. 核心流程

### 3.1 数据录入流程

1. 用户在小程序选择模块（采购/销售/库存）
2. 用户拍照或从相册选择鞋盒标签图片并上传
3. 后端调用豆包大模型解析图片，返回多条识别结果
4. 前端人工复核：允许修正模型字段，销售模块允许补充人工字段
5. 后端按模块规则处理：
   - 采购/库存：按 SKU_Code 聚合并累加写入飞书
   - 销售：明细流水，每条复核记录新增一条飞书记录
6. 返回同步结果：逐条成功/失败，并支持失败重试

### 3.2 库存查询流程

1. 用户进入“数据查询-库存查询”
2. 输入货号并发起查询
3. 后端从飞书库存表查询该货号记录
4. 后端按尺码（可选：按颜色+尺码）聚合数量
5. 前端展示结果列表

## 4. 模块字段与口径

### 4.1 字段来源约定

字段来源分为三类：
- 大模型解析：由豆包大模型从图片中提取
- 系统自动生成：由系统生成（例如聚合数量、图片附件、SKU_Code）
- 人工前端写入：由人工在复核页面补充/编辑

### 4.2 采购模块

- 大模型解析：供应商、货号、颜色、尺码
- 系统自动生成：
  - 数量：按 SKU_Code 聚合后计算
  - 对应图片：附件字段（同一张图片的多条记录可复用同一个 file_token）
  - SKU_Code：用于检索与去重的唯一键
- 写入口径：按 SKU_Code upsert，数量“写入时累加”

### 4.3 库存模块

- 大模型解析：货号、颜色、尺码
- 系统自动生成：
  - 数量：按 SKU_Code 聚合后计算
  - 对应图片：附件字段
  - SKU_Code：用于检索与去重的唯一键
- 写入口径：按 SKU_Code upsert，数量“写入时累加”

### 4.4 销售模块（明细流水）

- 大模型解析：货号、颜色、尺码
- 人工前端写入：数量、金额(元)、支付方式（单选）、备注
- 系统自动生成：对应图片（附件字段）
- 写入口径：每条记录均新增一条流水记录（不使用 SKU_Code 作为唯一键）

## 5. 交互与体验要点

- 人工复核页面支持逐条编辑、删除
- 同步结果支持逐条状态展示与失败原因展示
- 部分成功时明确提示成功/失败数量，并提供“重试失败项”入口
